Fraud Detection Pipeline (MVP)

Python 3.10 FastAPI Scikit-learn Cloud Deployment

Overview

This project is a Proof of Concept (PoC) demonstrating how to take a Machine Learning model from "Research" to "Production." Instead of using static datasets, I built a system that simulates user behavior, trains a model, and serves predictions via a public API.

Key Achievement: Transformed a notebook-based ML model into a production-ready web service with real-time inference capabilities.

Tech Stack

Core

Python 3.10

ML Engine

Scikit-learn (Logistic Regression), Pandas, NumPy

Backend

FastAPI, Uvicorn, Pydantic (Data Validation)

Frontend

HTML5, CSS3, Vanilla JavaScript (Fully AI-based)

Deployment

Render Cloud (PaaS), Git/GitHub

Key Technical Features

Synthetic Data Engineering

  • Since real financial data is private (PII), I wrote a Data Simulator to generate 1,000+ user records.
  • Injected specific "Fraud Patterns" (e.g., login spikes > 85/day) to test the model's ability to detect outliers vs. normal behavior.
  • Implemented imbalanced dataset techniques to simulate real-world fraud scenarios (99:1 normal-to-fraud ratio).

API Architecture

  • Implemented RESTful endpoints (POST/GET) with strict type validation using Pydantic.
  • Integrated CORS middleware to allow secure communication between the frontend and cloud backend.
  • Built automatic API documentation using FastAPI's built-in Swagger UI.
  • Structured endpoints for scalability: /predict, /health, /model-info

Model Persistence

  • Used joblib to serialize ("freeze") the trained model, allowing the server to restart without losing intelligence.
  • Implemented version control for model updates without downtime.

Operational Maturity

  • Includes a SQLite Database to log every prediction request for audit history.
  • Includes a Test Suite to verify logic before deployment.
  • Implemented error handling and logging for production reliability.

Challenges & Learnings

The "Imbalanced Data" Problem

I learned that 99% accuracy is misleading in fraud detection. When 99% of transactions are legitimate, a model that predicts "everything is legitimate" would achieve 99% accuracy but catch zero fraud!

Solution: I implemented a Confusion Matrix to track Precision and Recall instead of just accuracy. This revealed the true performance: how many frauds we caught vs. how many false alarms we triggered.

Deployment Journey

Navigated the difference between localhost and 0.0.0.0 bindings when moving from a local environment to the Render cloud. Had to configure environment variables properly and ensure the application listens on the correct port.

Frontend-Backend Integration

Initially struggled with CORS errors when the frontend tried to communicate with the deployed API. Resolved by adding proper CORS middleware configuration in FastAPI.
