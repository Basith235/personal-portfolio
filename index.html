AI Fraud Detection Pipeline (MVP)

Python 3.10 FastAPI Scikit-learn Cloud Render

SYSTEM OVERVIEW

This project is a Proof of Concept (PoC) demonstrating how to take a Machine Learning model from isolated research to a live production environment. Instead of relying on static CSV files, the system simulates user behavior, trains a logic model, and serves predictions via a public REST API.

> KEY_ACHIEVEMENT: Successfully transformed a local notebook-based ML model into a cloud-hosted web service capable of real-time inference.

ARCHITECTURE & STACK

ML ENGINE

Scikit-learn (Logistic Regression), Pandas, NumPy

BACKEND

FastAPI, Uvicorn, Pydantic (Data Validation)

PERSISTENCE

SQLite DB for audit logging, Joblib for model freezing.

CORE MODULES

Synthetic Data Generation

  • Developed a Data Simulator to generate 1,000+ mock user records, bypassing PII restrictions of real financial data.
  • Programmed specific "Fraud Patterns" (e.g., login velocity > 85/day) to train outlier detection.
  • Implemented a realistic 99:1 normal-to-fraud ratio to simulate real-world class imbalance.

API & Network Infrastructure

  • Exposed RESTful endpoints utilizing strict Pydantic type validation.
  • Configured CORS middleware to ensure secure cross-origin requests from front-end clients.
  • Built standard endpoints: /predict, /health, and /model-info.

Operational Integrity

  • Utilized joblib to serialize the trained model state, preventing data loss on server restart.
  • Integrated a SQLite ledger to track and audit every incoming prediction request.

DEBUG LOG & LEARNINGS

ERR: "The Imbalanced Data Paradox"

Initial testing revealed that a 99% accuracy metric is highly deceptive in fraud scenarios. Because 99% of generated data was benign, a model trained to simply output "False" for everything achieved 99% accuracy while catching zero actual fraud cases.

PATCH: Shifted evaluation metrics to a Confusion Matrix, tracking Precision and Recall instead to measure actual threat detection rates versus false-positive alarms.

ERR: "Cloud Binding Failures"

Encountered network binding errors transitioning from local dev to the Render cloud platform. Resolved by reconfiguring environment variables and correcting the host listener from localhost to 0.0.0.0.
